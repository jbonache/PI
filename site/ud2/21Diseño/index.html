    2.1. Disseny

    2.1. Fase de Disseny del Projecte

    En la Fase de Disseny del projecte, es desenvolupa una estructura sòlida que servirà de base per a la resta del desenvolupament. Esta fase se centra en la creació de diagrames i models que descriuen com els diferents components del sistema interactuen entre si i amb els usuaris.

    El disseny del projecte inclou:

    • Arquitectura tècnica
    • Diagrames UML
    • Experiència d’usuari (UX/UI)
    • Prototips
    • Documentació del disseny

    Aquests elements són clau per a garantir que el disseny del projecte siga coherent i complisca amb els objectius establerts.

    🎯 Objectius

    • Dissenyar l’arquitectura tècnica del projecte.
    • Crear diagrames UML basats en els requisits del projecte.
    • Desenvolupar interfícies intuïtives i atractives centrades en una experiència d’usuari fluïda i eficient.
    • Assegurar la coherència del disseny amb els objectius del projecte.

    🧩 Part 1: Clarificació de Requisits

    Es revisen i aclareixen els requisits específics del projecte perquè tot l’equip tinga una comprensió clara del que s’espera del sistema. Això és essencial per a dissenyar una solució que complisca amb les necessitats i expectatives dels usuaris i altres parts interessades.

    📐 Part 2: Creació de Diagrames UML

    Es desenvolupen diagrames UML per a representar l’arquitectura tècnica i les interaccions del sistema.

    Tipus de Diagrames

    • Diagrames de casos d’ús
    • Diagrames de classes
    • Diagrames de seqüències
    • Diagrames de components

    En este projecte, es fa especial èmfasi en:

    • Diagrama de casos d’ús: descriu com els actors interactuen amb el sistema per a aconseguir els seus objectius.
    • Diagrama de components: mostra l’estructura de l’aplicació i les relacions entre els seus diferents components.

    ![diagrama de casos de ús exemple.png]

    Exemple de diagrama de casos d’ús

    Un exemple clàssic d’un diagrama de casos d’ús per a una aplicació de comerç electrònic podria incloure diversos actors i les seues interaccions amb el sistema. Ací tens una descripció simple:

    Actors:

    • Client
    • Administrador

    Casos d’Ús Principals:

    • Client:
    • Registrar-se i iniciar sessió.
    • Cercar productes.
    • Afegir productes al carret.
    • Realitzar pagament.

    • Consultar estat de la comanda.

    • Administrador:

    • Gestionar inventari (afegir, eliminar productes).
    • Veure ordres de compra.
    • Gestionar comptes d’usuaris.

    Este diagrama modelaria com els usuaris (Client, Administrador) interactuen amb el sistema per a completar les seues tasques, mostrant les funcions principals del sistema relacionades amb cada actor.

    Codi PlantUML per a visualitzar el diagrama de casos d’ús

    Copia i pega el següent codi en un editor de text i guarda’l amb l’extensió .pu. Pots usar eines online com PlantUML Online Server per a visualitzar el diagrama.

    @startuml
left to right direction
actor Cliente
actor Administrador

rectangle "Sistema" {
  Cliente -- (Registrarse e iniciar sesión)
  Cliente -- (Buscar productos)
  Cliente -- (Añadir productos al carrito)
  Cliente -- (Realizar pago)
  Cliente -- (Consultar estado del pedido)

  Administrador -- (Gestionar inventario)
  Administrador -- (Ver órdenes de compra)
  Administrador -- (Gestionar cuentas de usuarios)
}
@enduml

    Exemple de diagrama de components

    Un diagrama de components mostra com els diferents components d’un sistema interactuen entre si per a complir amb els requisits del projecte. Ací tens un exemple simple d’un diagrama de components per a un sistema de client i servidor:

    Components:

    • Client
    • Servidor
    • Base de Dades

    Interaccions:

    • El client envia una sol·licitud al servidor.
    • El servidor processa la sol·licitud i envia una resposta al client.
    • El servidor accedeix a la base de dades per a recuperar o emmagatzemar informació.

    Codi PlantUML per a visualitzar el diagrama de components

    Copia i pega el següent codi en un editor de text i guarda’l amb l’extensió .pu. Pots usar eines online com PlantUML Online Server per a visualitzar el diagrama.

    @startuml

package "Cliente" {
  [Interfaz de Usuario]
}

package "Servidor" {
  [Controlador]
  [Modelo]
  [Vista]
}

package "Base de Datos" {
  [BD]
}

[Interfaz de Usuario] --> [Controlador]
[Controlador] --> [Modelo]
[Modelo] --> [BD]

@enduml

    ![diagrama-de-componentes-ejemplo-bbdd.png]

    Per a aconseguir aquest diagrama de components, copia i pega el següent codi en un editor de text i guarda’l amb l’extensió .pu per a visualitzar el diagrama de components amb PlantUML.

    @startuml

package "Cliente" {
  [Interfaz de Usuario]
}

package "Servidor" {
  [Controlador]
  [Modelo]
  [Vista]

  [BD]

[Interfaz de Usuario] --> [Controlador]

[Controlador] --> [Modelo]

[Modelo] --> [Base de Datos]

@enduml

    Part 3: Definició de l’Experiència de l’Usuari (UX/UI)

    En esta etapa, es dissenya la interfície d’usuari per a proporcionar una experiència d’usuari intuïtiva i atractiva. Això implica la creació de prototips de les interfícies d’usuari per a validar el disseny i la usabilitat del sistema. El disseny de la interfície d’usuari ha d’estar centrat en la funcionalitat i l’estètica per a garantir una experiència d’usuari fluïda i eficient.

    Dissenya interfícies intuïtives i atractives, centrades en proporcionar una experiència d’usuari fluïda i eficient, tenint en compte tant la funcionalitat com l’estètica.

    Aquests elements són clau per a garantir que el disseny del projecte siga coherent i complisca amb els objectius establerts.

    Arquitectura del Sistema

    Defineix com els components principals del sistema interactuaran entre si per a complir amb els requisits especificats.

    Creació de Diagrames Essencials

    Inclou diagrames de classes per a l’estructura de dades, diagrames de flux de dades per al moviment d’informació, i models entitat-relació per a la base de dades.
