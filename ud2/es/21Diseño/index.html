    2.1. Fase de Diseño del Proyecto

    En la Fase de Diseño del proyecto, se desarrolla una estructura sólida que servirá de base para el resto del desarrollo. Esta fase se centra en la creación de diagramas y modelos que describen cómo los diferentes componentes del sistema interactúan entre sí y con los usuarios.

    El diseño del proyecto incluye:

    • Arquitectura técnica
    • Diagramas UML
    • Experiencia del usuario (UX/UI)
    • Prototipos
    • Documentación del diseño

    Estos elementos son clave para garantizar que el diseño del proyecto sea coherente y cumpla con los objetivos establecidos.

    🎯 Objetivos

    • Diseñar la arquitectura técnica del proyecto.
    • Crear diagramas UML basados en los requisitos del proyecto.
    • Desarrollar interfaces intuitivas y atractivas centradas en una experiencia de usuario fluida y eficiente.
    • Asegurar la coherencia del diseño con los objetivos del proyecto.

    🧩 Parte 1: Clarificación de Requisitos

    Se revisan y aclaran los requisitos específicos del proyecto para que todo el equipo tenga una comprensión clara de lo que se espera del sistema. Esto es esencial para diseñar una solución que cumpla con las necesidades y expectativas de los usuarios y otras partes interesadas.

    📐 Parte 2: Creación de Diagramas UML

    Se desarrollan diagramas UML para representar la arquitectura técnica y las interacciones del sistema.

    Tipos de Diagramas

    • Diagramas de casos de uso
    • Diagramas de clases
    • Diagramas de secuencias
    • Diagramas de componentes

    En este proyecto, se hace especial énfasis en:

    • Diagrama de casos de uso: describe cómo los actores interactúan con el sistema para lograr sus objetivos.
    • Diagrama de componentes: muestra la estructura de la aplicación y las relaciones entre sus distintos componentes.

    **Ejemplo de diagrama de casos de uso:

    ![diagrama de casos de uso ejemplo.png]

    Ejemplo de diagrama de casos de uso

    Un ejemplo clásico de un diagrama de casos de uso para una aplicación de comercio electrónico podría incluir varios actores y sus interacciones con el sistema. Aquí tienes una descripción simple:

    Actores:

    • Cliente
    • Administrador

    Casos de Uso Principales:

    • Cliente:
    • Registrarse e iniciar sesión.
    • Buscar productos.
    • Añadir productos al carrito.
    • Realizar pago.

    • Consultar estado del pedido.

    • Administrador:

    • Gestionar inventario (añadir, eliminar productos).
    • Ver órdenes de compra.
    • Gestionar cuentas de usuarios.

    Este diagrama modelaría cómo los usuarios (Cliente, Administrador) interactúan con el sistema para completar sus tareas, mostrando las funciones principales del sistema relacionadas con cada actor.

    Código PlantUML para visualizar el diagrama de casos de uso

    Copia y pega el siguiente código en un editor de texto y guárdalo con la extensión .pu. Puedes usar herramientas online como PlantUML Online Server para visualizar el diagrama.

    @startuml
left to right direction
actor Cliente
actor Administrador

rectangle "Sistema" {
  Cliente -- (Registrarse e iniciar sesión)
  Cliente -- (Buscar productos)
  Cliente -- (Añadir productos al carrito)
  Cliente -- (Realizar pago)
  Cliente -- (Consultar estado del pedido)

  Administrador -- (Gestionar inventario)
  Administrador -- (Ver órdenes de compra)
  Administrador -- (Gestionar cuentas de usuarios)
}
@enduml

    Ejemplo de diagrama de componentes

    Un diagrama de componentes muestra cómo los diferentes componentes de un sistema interactúan entre sí para cumplir con los requisitos del proyecto. Aquí tienes un ejemplo simple de un diagrama de componentes para un sistema de cliente y servidor:

    Componentes:

    • Cliente
    • Servidor
    • Base de Datos

    Interacciones:

    • El cliente envía una solicitud al servidor.
    • El servidor procesa la solicitud y envía una respuesta al cliente.
    • El servidor accede a la base de datos para recuperar o almacenar información.

    Código PlantUML para visualizar el diagrama de componentes

    Copia y pega el siguiente código en un editor de texto y guárdalo con la extensión .pu. Puedes usar herramientas online como PlantUML Online Server para visualizar el diagrama.

    @startuml

package "Cliente" {
  [Interfaz de Usuario]
}

package "Servidor" {
  [Controlador]
  [Modelo]
  [Vista]
}

package "Base de Datos" {
  [BD]
}

[Interfaz de Usuario] --> [Controlador]
[Controlador] --> [Modelo]
[Modelo] --> [BD]

@enduml

    ![diagrama-de-componentes-ejemplo-bbdd.png]

    Para conseguir este diagrama de componentes, copia y pega el siguiente código en un editor de texto y guárdalo con la extensión .pu para visualizar el diagrama de componentes con PlantUML.

    @startuml

package "Cliente" {
  [Interfaz de Usuario]
}

  [Modelo]

[Controlador] --> [Modelo]

@enduml

    Parte 3: Definición de la Experiencia del Usuario (UX/UI)

    En esta etapa, se diseña la interfaz de usuario para proporcionar una experiencia de usuario intuitiva y atractiva. Esto implica la creación de prototipos de las interfaces de usuario para validar el diseño y la usabilidad del sistema. El diseño de la interfaz de usuario debe centrarse en la funcionalidad y la estética para garantizar una experiencia de usuario fluida y eficiente.

    Diseña interfaces intuitivas y atractivas, centradas en proporcionar una experiencia de usuario fluida y eficiente, tomando en cuenta tanto la funcionalidad como la estética.

    Estos elementos son clave para garantizar que el diseño del proyecto sea coherente y cumpla con los objetivos establecidos.

    Seguir leyendo sobre UX/UI [https://smascarelleljust.github.io/PI/2.%20Dise%C3%B1o/2.3.%20UX/Definici%C3%B3n%20de%20la%20Experiencia%20del%20Usuario/]

    Arquitectura del Sistema

    Define cómo los componentes principales del sistema interactuarán entre sí para cumplir con los requisitos especificados.

    Creación de Diagramas Esenciales

    Incluye diagramas de clases para la estructura de datos, diagramas de flujo de datos para el movimiento de información, y modelos entidad-relación para la base de datos.
