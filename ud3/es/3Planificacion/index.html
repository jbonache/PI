    1️⃣ Análisis Inicial del Proyecto

    En esta fase, debes analizar el alcance del proyecto, definir tus objetivos y establecer una base sólida para la planificación. Se recomienda que trabajes en equipos o individualmente y respondas a las siguientes secciones.

    📌 1.1 Definición del Proyecto

    • Objetivo principal: ¿Cuál es el propósito del proyecto? ¿Qué problema soluciona o qué necesidad cubre?
    • Usuarios o clientes objetivo: ¿Quién usará la aplicación? (Ejemplo: estudiantes, empresas, particulares)
    • Funcionalidades esenciales: ¿Qué debe ser capaz de hacer la aplicación o el sistema? (Ejemplo: iniciar sesión, gestionar reservas, enviar notificaciones)
    • Tecnologías previstas: ¿Con qué lenguajes, frameworks o herramientas se desarrollará?
    • Requisitos iniciales: ¿Hay alguna restricción técnica o funcional que se debe considerar?

    Ejemplo de respuesta:
    "Nuestro proyecto es una aplicación móvil para la gestión de eventos, permitiendo a los usuarios registrarse, crear eventos y enviar invitaciones. Estará dirigido a organizadores de conferencias y usará Flutter con Firebase."

    📌 1.2 Identificación de Módulos Relacionados

    El proyecto se desarrollará integrando conocimientos de diferentes módulos del ciclo formativo DAM. Debes identificar qué módulos están implicados y justificar su relación con el proyecto.

    • Programación: Desarrollo de la lógica del software.
    • Bases de Datos: Gestión del almacenamiento y recuperación de datos.
    • Interfaces Gráficas: Diseño de UI/UX para mejorar la experiencia de usuario.
    • Sistemas de Gestión Empresarial: Integración de metodologías y herramientas de gestión de proyectos.
    • Servicios y Procesos: Uso de APIs o servicios en segundo plano.

    Ejemplo de respuesta:
    "Nuestro sistema de reservas necesitará una base de datos en MySQL, por lo que el módulo de Bases de Datos será clave. También implementaremos una interfaz en JavaFX (Interfaces Gráficas) y lógica en Java (Programación).”

    📌 1.3 Hitos y Entregables Iniciales

    En este apartado, debes identificar los hitos clave del desarrollo y qué entregables resultarán de cada uno. Se recomienda definir 4 o 5 hitos principales.

    Hito Descripción Entregable
    Hito 1: Especificación de requisitos Definir requisitos funcionales y no funcionales Documento de requisitos del sistema
    Hito 2: Diseño del sistema Creación de diagramas de arquitectura y UI Bocetos y diagramas UML
    Hito 3: Desarrollo del prototipo Implementación básica con funciones esenciales Versión inicial del software
    Hito 4: Pruebas y validación Evaluar errores y optimizar rendimiento Informe de pruebas y correcciones
    Hito 5: Entrega final Preparar la versión definitiva del proyecto Software terminado y documentación

    Ejemplo de respuesta:
    "Para la primera entrega, crearemos un documento de requisitos con los aspectos clave de la plataforma. Luego, trabajaremos en el diseño de interfaces y en un prototipo funcional con las funcionalidades básicas."

    📌 1.4 Restricciones y Recursos

    Debes analizar los factores que pueden condicionar el desarrollo del proyecto.

    📌 Factores a evaluar:

    • Tiempo disponible: ¿Cuánto tiempo se prevé para cada fase?
    • Recursos tecnológicos: ¿Qué herramientas y plataformas se utilizarán? (Ejemplo: GitHub, Trello, Firebase)
    • Limitaciones técnicas: ¿El proyecto tiene restricciones de compatibilidad o rendimiento?
    • Dependencias externas: ¿Es necesario usar APIs, SDKs o integraciones con otros sistemas?

    Ejemplo de respuesta:
    "El desarrollo se realizará en 4 semanas. Usaremos Trello para la gestión de tareas y GitHub para el control de versiones. Nuestra aplicación necesitará integrarse con Google Maps, por lo que debemos revisar su API."

    🎯 Resultado Final

    Al finalizar esta fase, habrás elaborado un documento con:

    1. Definición clara del proyecto.
    2. Relación con los módulos del ciclo.
    3. Lista de hitos y entregables.
    4. Restricciones y recursos necesarios.

    Esto servirá como base para la planificación detallada del proyecto en las siguientes fases.
