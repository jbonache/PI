    2.2.2. Diagrama de Componentes

    Objetivo

    Diseñar la arquitectura técnica de una aplicación específica del proyecto de cada grupo utilizando diagramas UML basados en sus requisitos únicos.

    Parte 1: Clarificación de Requisitos

    • Cada grupo revisa y aclara los requisitos específicos de su propio proyecto.
    • Discutir cualquier ambigüedad o falta de claridad en los requisitos.
    • Asegurarse de que todos los miembros del grupo tengan una comprensión clara de lo que se espera del sistema.
    • Identificar los componentes clave de la aplicación y cómo interactúan entre sí.
    • Definir las interfaces entre los diferentes componentes.

    Parte 2: Creación de Diagramas UML

    Casos de Uso
    • Identificar los actores principales y los casos de uso de la aplicación.
    • Desarrollar un diagrama de casos de uso que muestre cómo los actores interactúan con la aplicación para lograr sus objetivos.
    • Incluir descripciones breves de cada caso de uso para aclarar su propósito y funcionalidad.
    Diagrama de Componentes
    • Identificar los componentes principales de la aplicación y cómo se relacionan entre sí.
    • Desarrollar un diagrama de componentes que muestre la estructura de la aplicación y cómo los diferentes componentes interactúan.
    • Incluir interfaces y dependencias entre los componentes para reflejar las relaciones clave.
    • Asegurarse de que el diagrama de componentes sea claro y fácil de entender para todos los miembros del grupo.
    • Revisar y refinar el diagrama de componentes según sea necesario para reflejar con precisión la arquitectura técnica de la aplicación.

    Materiales Necesarios

    • Papeles grandes o acceso a software de diagramación UML.
    • Marcadores, lápices o acceso a herramientas digitales si se prefiere el diseño digital.
    • Diagrama de componentes [https://creately.com/blog/es/diagramas/tutorial-de-diagrama-de-componentes/]
    • Plantext [https://www.planttext.com/]
    • PlantUML [https://plantuml.com/es/]

    Diagrama de Componentes

    Ejemplo de Diagrama de Componentes

    @startuml
package "Sistema de Reservas para Restaurantes" {
    [Interfaz de Usuario] as UI
    [Sistema de Gestión de Reservas] as Manager
    [Base de Datos] as DB

    package "Módulo de Reservas" {
        [Crear Reservas]
        [Modificar Reservas]
        [Cancelar Reservas]
        Manager --> [Crear Reservas]
        Manager --> [Modificar Reservas]
        Manager --> [Cancelar Reservas]
    }

    package "Control de Mesas" {
        [Estado de Mesas]
        [Capacidad de Mesas]
        [Asignación Automática]
        Manager --> [Estado de Mesas]
        Manager --> [Capacidad de Mesas]
        Manager --> [Asignación Automática]
    }

    package "Notificaciones" {
        [Enviar Confirmaciones]
        [Enviar Recordatorios]
        [Alertas de Cancelación]
        Manager --> [Enviar Confirmaciones]
        Manager --> [Enviar Recordatorios]
        Manager --> [Alertas de Cancelación]
    }

    package "Gestión de Usuarios" {
        [Clientes]
        [Administradores]
        UI --> [Clientes]
        UI --> [Administradores]
    }

    package "Reportes" {
        [Historial de Reservas]
        [Tendencias de Ocupación]
        Manager --> [Historial de Reservas]
        Manager --> [Tendencias de Ocupación]
    }

    [Clientes] ..> UI : Utiliza
    [Administradores] ..> UI : Utiliza
    UI ..> Manager : Interactúa
    Manager ..> DB : Almacena/Recupera Datos
}

@enduml

    Explicación del Diagrama:

    • Interfaz de Usuario (UI): Punto de interacción para clientes y administradores.
    • Sistema de Gestión de Reservas (Manager): Núcleo central que maneja la lógica de negocio, incluyendo la creación, modificación y cancelación de reservas.
    • Base de Datos (DB): Almacena toda la información relevante como detalles de las reservas, estado de las mesas, información de los usuarios, etc.
    • Módulos Funcionales: Cada paquete representa una funcionalidad específica del sistema, desde gestionar reservas hasta enviar notificaciones y generar reportes.
